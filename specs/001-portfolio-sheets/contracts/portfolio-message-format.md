# Contract: 持倉區塊 Telegram 訊息格式

## 插入位置

持倉區塊插入在**漲跌行之後、RSI 行之前**（若有持倉）：

```
{ARROW} *{SYMBOL}* {DISPLAY_NAME}
   現價: `{PRICE} TWD`   昨收: `{PREV_CLOSE}`
   漲跌: `{SIGN}{CHANGE}` \({SIGN}{PCT}%\)
   ─── 持倉 ───                            ← 僅持有時顯示
   持股: `{SHARES}`   成本: `{AVG_COST}` \({COST_PCT}%\)
   損益: `{PNL_SIGN}{PNL} TWD`
   RSI\(14\): `{RSI}` {RSI_TAG}            ← 原有欄位繼續
   ...
```

## 欄位說明

| 佔位符 | 說明 | 範例 |
|--------|------|------|
| `{SHARES}` | 持股數（含千分位） | `1,000` |
| `{AVG_COST}` | 平均成本，保留 2 位小數 | `820.00` |
| `{COST_PCT}` | 現價相對成本漲跌幅（%），含 +/- 號 | `+9.15` / `-3.20` |
| `{PNL_SIGN}` | 損益正負號 | `+` / `-` / 空 |
| `{PNL}` | 未實現損益（含千分位，保留 0 位小數） | `75,000` / `-12,000` |

## MarkdownV2 Escape 規則

- `{COST_PCT}` 的 `.` 與 `+`/`-` 必須 escape（用 `_escape_md()`）
- `{PNL}` 的 `+`/`-` 與 `,` 必須 escape
- `─── 持倉 ───` 中的 `─` 為 Unicode 破折號，不需 escape

## 條件顯示

- `avg_cost is None` 或 `shares is None` → 整個持倉區塊（含 `─── 持倉 ───`）不顯示
- `unrealized_pnl is None` → 損益行不顯示（成本行仍顯示）
- 美股（`market == 'US'`）→ 持倉區塊永不顯示

## 實際範例

```
🔺 *2330* 台積電
   現價: `895\.00 TWD`   昨收: `885\.00`
   漲跌: `\+10\.00` \(\+1\.13%\)
   ─── 持倉 ───
   持股: `1,000`   成本: `820\.00` \(\+9\.15%\)
   損益: `\+75,000 TWD`
   RSI\(14\): `62\.3`
   均線: `MA20:880↑  MA50:850↑`
```

## 損益為負範例

```
🔻 *2454* 聯發科
   現價: `780\.00 TWD`   昨收: `790\.00`
   漲跌: `\-10\.00` \(\-1\.27%\)
   ─── 持倉 ───
   持股: `500`   成本: `850\.00` \(\-8\.24%\)
   損益: `\-35,000 TWD`
```
